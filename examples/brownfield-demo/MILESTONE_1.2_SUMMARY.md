### 4. Xtask Integration ✅

**Location**: `examples/brownfield-demo/xtask/`

**Key Content** (xtask/src/main.rs):

```rust
use anyhow::Result;
use clap::{Parser, Subcommand};
use rust_iac_xtask_core::{commands, InitMode};

#[derive(Parser)]
#[command(name = "xtask")]
struct Cli {
    #[command(subcommand)]
    command: Commands,
}

#[derive(Subcommand)]
enum Commands {
    Init {
        #[arg(long, default_value = "brownfield")]
        mode: String,
    },
    Selftest,
}

fn main() -> Result<()> {
    let cli = Cli::parse();

    match cli.command {
        Commands::Init { mode } => {
            let init_mode: InitMode = mode.parse()?;
            commands::init::init(init_mode, None)?;
            // ... helpful next steps
        }
        Commands::Selftest => {
            commands::selftest::selftest(None)?;
        }
    }
    Ok(())
}
```

**Design Pattern**:
- Minimal wrapper around `rust_iac_xtask_core`
- Uses path dependency: `rust_iac_xtask_core = { path = "../../../rust_iac_xtask_core" }`
- Easily customizable per project
- Clean separation of concerns

## Friction Points Discovered

### 1. Workspace Configuration ⚠️

**Issue**: Need to exclude examples from parent workspace

**Solution**: Added to root `Cargo.toml`:
```toml
[workspace]
exclude = [
  "rust_iac_xtask_core",
  "examples/brownfield-demo",
]
```

**Impact**: Minor - one-time configuration
**Severity**: Low
**Documentation**: Added to troubleshooting section

### 2. Path Dependencies 📝

**Issue**: Relative path dependencies can be brittle

**Current**:
```toml
rust_iac_xtask_core = { path = "../../../rust_iac_xtask_core" }
```

**Future Options**:
1. Publish to crates.io: `rust_iac_xtask_core = "0.1.0"`
2. Git dependency: `rust_iac_xtask_core = { git = "...", tag = "v0.1.0" }`

**Impact**: Medium - affects adoption
**Severity**: Low
**Documentation**: Included migration guide in README

### 3. Axum Route Syntax Change 🔧

**Issue**: Axum 0.8 changed from `:param` to `{param}` syntax

**Fix**: Updated route from `/items/:id` to `/items/{id}`

**Impact**: Low - documentation needed
**Severity**: Low
**Documentation**: Fixed in code

### 4. No Friction! ✅

**Positive Findings**:

```bash

cd examples/brownfield-demo

# 2. Build everything
cargo build

# 4. Initialize Rust IaC (in original terminal after stopping server)
cargo run -p xtask -- init --mode=brownfield

# 5. Verify with selftest
cargo run -p xtask -- selftest

# 6. Review generated files
cat RUST_IAC.toml

cat policy/example.rego
cat .llm/contextpack.yaml

# 7. Run server tests
cargo test -p server

# 8. Run core library tests
cd ../../rust_iac_xtask_core
cargo test
```

### For Brownfield Project Adoption

```bash
# 1. Copy the xtask directory to your project
cp -r examples/brownfield-demo/xtask /path/to/your/project/

# 2. Update your project's Cargo.toml workspace
# Add: members = ["xtask"]

# 4. Run init

cargo run -p xtask -- selftest
```
