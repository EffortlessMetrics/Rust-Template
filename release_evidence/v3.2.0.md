## Known Limitations

### Windows File Locking

**Issue:** On Windows, cargo may fail to clean build artifacts between test runs due to file locking:
```
error: failed to remove file `target\debug\xtask.exe`
Caused by: Access is denied. (os error 5)
```

**Workaround:** Close any processes holding locks (e.g., terminals, editors) or restart the terminal session before running `cargo clean`.

**Impact:** Does not affect functionality, only incremental builds. All tests pass when build succeeds.

### Non-Kernel ACs (Future Features)

**Scope:** 5 unknown ACs are intentionally classified as:
- **Manual verification** (AC-TPL-AGENT-SKILLS): Skills definitions exist, validated manually
- **Unit tests** (AC-PLT-014, AC-TPL-GRAPH-AC-HAS-TEST): Validated in selftest step 7
- **Future features** (AC-TPL-TASKS-CREATE-CLI, AC-TPL-TASKS-UPDATE-CLI): Not blocking v3.2.0

**Rationale:** These ACs either:
1. Have non-BDD test coverage (unit tests, manual verification)
2. Are intentionally deferred to future releases
3. Have clear classification with documented reasoning

**Tracking:** All unknown ACs documented in `docs/feature_status.md` with classification rationale.

### Failing Kernel ACs

**Count:** 14 failing ACs (25.5% of total)

**Categories:**

**Impact:** These failures are **known implementation gaps** tracked with BDD scenarios. They do not block release but provide clear roadmap for v3.3.0.
