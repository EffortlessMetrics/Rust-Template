## 2024-03-03 - Prevent FOUC and Enhance Accessibility in Dynamic UIs
**Learning:** Adding dynamic initial classes (e.g. `.active`) and `aria-pressed` states on the client side using `DOMContentLoaded` can cause a Flash of Unstyled Content (FOUC) and an initial state of confusion for screen readers. In Server-Side Rendered (SSR) environments like Maud, the initial state should be baked in on the server. Furthermore, adding `aria-live="polite"` to dynamic content containers (like a data table responding to filter/search events) ensures screen readers reliably announce updates.
**Action:** When adding state-dependent styling and accessibility attributes to interactive components in SSR pages, explicitly add the default state attributes (e.g., `aria-pressed="true"`, `.active`) directly in the server template. Use `aria-live` regions for any content area that updates asynchronously without a page reload.
