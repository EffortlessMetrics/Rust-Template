## PHASE 3: PERFORMANCE & OPTIMIZATION

### 3.1 Parallelize BDD Test Execution [HIGH-IMPACT]

**Current**: 8-15 min sequential execution (max_concurrent_scenarios=1)
**Target**: 2-3 min with 4-6 parallel scenarios
**Files to Modify**:
- `crates/acceptance/tests/acceptance.rs` (World setup)
- `crates/acceptance/src/lib.rs` (SPEC_ROOT handling)

**Issue**: Global SPEC_ROOT environment variable causes cross-test interference

**Solution**: Per-test isolation
```rust
// Before:
lazy_static! {
    static ref SPEC_ROOT: PathBuf = std::env::var("SPEC_ROOT")
        .map(PathBuf::from)
        .unwrap_or_else(|_| PathBuf::from("."));
}

// After:
fn world_spec_root() -> PathBuf {
    thread_local! {
        static SPEC_ROOT: PathBuf = std::env::var("SPEC_ROOT")
            .map(PathBuf::from)
            .unwrap_or_else(|_| PathBuf::from("."));
    }
    SPEC_ROOT.with(|p| p.clone())
}

// In acceptance.rs
World::cucumber()
    .max_concurrent_scenarios(6)  // Now safe to parallelize
    .before(|feature, rule, scenario, world| { ... })
```

**Effort**: 3-4 hours (careful refactoring)
**Priority**: HIGH (50-70% CI speedup)
