# Feature Status Notes

Some acceptance criteria (ACs) in the template are placeholders that you may or may not adopt in your service:

- `AC-TPL-*` entries that are not part of your kernel can remain unimplemented while you onboard. Treat them as "Template / not yet adopted" rather than failures.
- When `ac-status` reports these ACs as unknown or unmapped, document which ones you plan to keep and which ones you will archive.
- Once you intentionally adopt an AC, link it from `specs/spec_ledger.yaml` and add scenarios or unit tests so it moves out of the placeholder bucket.
