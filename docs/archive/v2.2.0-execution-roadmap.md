# v2.2.0 Execution Roadmap

**Status**: ✅ Complete (Released 2025-11-17)
**Strategy**: Small, sequential PRs with continuous validation
**Sequence**: DB test → GRPC test → LLM/DX → OTLP (optional)

---

## Why This Order?

1. **DB test first** - Fast win, proves persistence layer, deepens infra-as-code
2. **GRPC test second** - Proves API layer, completes adapter coverage
3. **LLM/DX third** - Improves ergonomics without touching core code
4. **OTLP last** - Can punt to v2.3.0 if it becomes a time sink

This de-risks OTLP and ensures v2.2.0 ships even if OTLP is harder than expected.

## Quick Reference Commands

```bash
# Start new PR branch
git checkout -b feat/db-adapter-integration-test

# Validation loop (run after each change)
cargo fmt --all
cargo clippy --workspace --all-targets -- -D warnings
cargo test --workspace
cargo xtask selftest

# Run ignored tests
cargo test -p adapters-db-sqlx -- --ignored
cargo test -p adapters-grpc --test smoke -- --ignored

# Build with OTLP (if implementing)
cargo build --features telemetry/otlp

# Tag and push release
git tag -a v2.2.0 -m "Rust IaC Template v2.2.0"
git push origin main --tags
```
