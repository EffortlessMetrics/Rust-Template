# v3.4.0 Release Plan: IDP-Ready Kernel Framing

## Theme

**"IDP-Ready"** – The kernel provides stable, machine-readable contracts for Internal Developer Platform integration.

## Scope

### Three Pillars of "IDP-Ready"

1. **IDP Snapshot Contract Frozen**
   - `cargo xtask idp-snapshot` emits stable JSON
   - `/platform/idp/snapshot` HTTP endpoint matches CLI output
   - `specs/platform_schema.yaml` defines response schema
   - ACs: `AC-TPL-IDP-SNAPSHOT`, `AC-TPL-IDP-SNAPSHOT-VALID-JSON`

2. **Docs Automation Mandatory**
   - `docs-check` validates all governance invariants
   - Version alignment enforced across 8+ files (`AC-PLT-009`)
   - Feature status header invariants (`AC-PLT-010`)
   - Doc index ↔ frontmatter bidirectional sync (`AC-PLT-DOC-INDEX-FRONTMATTER`)

3. **Environment Diagnostics Complete**
   - Centralized CI/non-interactive helpers
   - `AC-TPL-XTASK-NONINTERACTIVE` enforced for all DevEx commands
   - `cargo xtask doctor` provides comprehensive environment checks

## Guardrails

### What Will NOT Change

- Existing `/platform/*` endpoint paths and response structures
- `spec_ledger.yaml` schema (additive only)
- CLI command names and basic options
- Selftest gate structure (11 steps)

### Breaking Change Policy

- No removals of existing fields in JSON responses
- Deprecations announced one minor version before removal
- All changes validated against existing BDD scenarios
